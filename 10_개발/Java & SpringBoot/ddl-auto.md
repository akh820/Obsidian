#jpa
**주요 옵션:**
1. **create** - 기존 테이블 삭제 후 새로 생성 (데이터 날아감!)
2. **update** - 테이블 없으면 생성, 있으면 컬럼 **추가만** (삭제/수정 안 함)
3. **create-drop** - create + 앱 종료 시 테이블 삭제
4. **validate** - 테이블과 엔티티 일치 확인만 (변경 안 함)
5. **none** - 아무것도 안 함

**update의 동작:**

- 매번 실행 시 엔티티 확인
- 새 컬럼 있으면 **추가** ✅
- 컬럼 삭제/타입 변경은 **안 함** ❌

**권장:**

- 개발: `update`
- 운영: `validate` 또는 `none` (Flyway/Liquibase 사용)
